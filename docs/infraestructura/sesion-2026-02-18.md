## Bugs corregidos

### 🔴 Bug: `<button>` anidado en `UpsellDrinksModal`

**Síntoma:** Error en consola `In HTML, <button> cannot be a descendant of <button>. This will cause a hydration error.`

**Causa:** la tarjeta de bebida era un `<button>` (para seleccionar) que contenía otro `<button>` (el ✕ para quitar). HTML no permite botones anidados.

**Solución:** La tarjeta se convirtió en `<div role="button">` con `onClick` + `onKeyDown` para mantener accesibilidad. El botón ✕ quedó como hermano posicionado en absoluto, no como hijo.

```jsx
// ❌ Antes — HTML inválido
<button onClick={increment}>
    <button onClick={decrement}>✕</button>
</button>

// ✅ Ahora — estructura correcta
<div role="button" onClick={increment}>
    ...
</div>
<button onClick={(e) => { e.stopPropagation(); decrement(); }}>✕</button>
```

### 5. Flujo Completo de Domicilios (`DeliveryFlow.jsx`)

**Propósito:** Permitir que el cliente ingrese su barrio y verificar si tiene cobertura antes de mostrar el menú.

**Flujo:**
```
Home → PEDIR A DOMICILIO
    → DeliveryFlow (pantalla completa)
        → Input de barrio + autocomplete
        → checkCoverage()
            ├── Con cobertura → asigna sede → /menu?type=delivery
            └── Sin cobertura → NoCobertura
                    ├── Recoger en Tienda → Home
                    └── Ingresar otra dirección → vuelve al input
```

**Autocomplete:**
- Busca en `ALL_CALI_NEIGHBORHOODS` (~250 barrios)
- Cada sugerencia muestra indicador visual: 🟢 con cobertura / ⚪ sin cobertura
- Solo requiere 2 caracteres para activarse
- Máximo 8 sugerencias

**Puntos de acceso:**
1. Botón "PEDIR A DOMICILIO" en el Hero de la Home
2. "Prefiero pedir a domicilio" dentro del `LocationSelector` (selector de pickup)

### 6. Base de datos de zonas de cobertura (`lib/deliveryZones.js`)

**Propósito:** Fuente de verdad para las zonas de delivery. Sin dependencia de Google Maps.

**Arquitectura (2 capas):**

| Capa | Descripción | Tamaño |
|---|---|---|
| `ALL_CALI_NEIGHBORHOODS` | Todos los barrios de Cali (autocomplete) | ~250 barrios |
| `COBERTURA_LIBERTADORES` | Barrios con entrega desde Sede Libertadores | 41 barrios |
| `COBERTURA_CALDAS` | Barrios con entrega desde Sede Caldas | 27 barrios |
| **Total con cobertura** | | **68 barrios** |

**Zonas basadas en los mapas del cliente:**

| Sede | Zona geográfica |
|---|---|
| **Libertadores** | Centro-norte: Chipichape → Huandama, Cras 15–70 |
| **Caldas** | Sur-occidente: El Guabal → ICESI/Pance, Cras 28–100+ |

**Funciones exportadas:**
- `checkCoverage(neighborhood)` → `{ covered, sede_slug }` o `{ covered: false }`
- `getNeighborhoodSuggestions(query, limit)` → array con `{ name, covered, sede }`
- `COVERAGE_ZONES_DISPLAY` → datos para mostrar en UI
- `TOTAL_COVERAGE_BARRIOS` → conteo total

**Para agregar barrios:**
```js
// En COBERTURA_CALDAS o COBERTURA_LIBERTADORES (minúsculas, sin tilde):
const COBERTURA_CALDAS = new Set([
    ...
    'nuevo barrio',  // ← agregar aquí
]);
// Si el barrio no existe en ALL_CALI_NEIGHBORHOODS, agregarlo allí también.
```
